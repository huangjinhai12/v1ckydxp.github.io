查看文件基本信息

32位程序，只开启了canary保护，栈不可执行保护和地址随机化都没有开启。

分析程序基本行为

基本功能就是添加，打印和删除的功能，但是存储name的index 可以不局限于数字，也可以是字符串等。

IDA分析程序逻辑

Del_note()函数，在free之后将指针置为了0，所以这里不存在UAF。

index只是判断了是否大于10，没有判断下界，这里存在数组越界写，可以在比数组低的位置进行任意写。还判断输入的name是否是可打印字符，这里限制了name必须为可打印字符。

利用

note指针保存的地址属于bss段，由于可以往数组低地址写，可以改写got表，如我们可以改写printf函数的got地址为shellcode地址，下一次调用printf函数的时候就是调用的shellcode。

0x0804A060-0x804A020 = 64; 64/4 =16，所以输入index为-16就可以覆盖puts.got的地址覆盖为shellcode的地址。

shellcode的编写

这道题主要考的是shellcode 的编写，要全部属于可见字符。不太会汇编，这里是直接参考的学长的。
