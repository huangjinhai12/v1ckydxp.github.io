​ 距离上一次坐pwn题已经是五一了。五一之后被迫搞了几周的渗透，接着又要准备第五届tsctf，有赶上课设作业要开发一个微信小程序。感觉五月还没干啥就到六月了，对于我这个pwn新手来说，一个月不碰了，感觉好生疏，又不敢开始了，之前学的一些东西也感觉好多都忘了。前两周学长要毕业了，一起吃了个饭，感觉学长都好厉害，都拿到了大厂的offer，我也要拿大厂的offer。听学长的建议，开始刷leetcode吧代码基础练起来，但是写代码这种东西和我之前学pwn一样，开始都会上瘾。但是我两边都不想落下。索引今天鼓起勇气重新学pwn。

堆的unlink利用方式的原理我看了好久，我之前就只是看，没有动手去调，很多地点都理解不了。先把我自己的遇到的问题写在前面：

  1. got表和plt表分别保存的是函数的真正的执行地址、函数的got表地址。修改free@got指针的内容为put@plt。

文件基本信息

程序是动态链接，出了pie，其他保护全部开启。运行程序不会输出任何东西。把程序拖进ida查看逻辑。

程序逻辑分析

程序主要有4个功能：

  1. alloc：根据用户输出的size分配大小
  2. Edit：可以对指定的索引进行编辑，输入字符串的大小可以控制，这里可以造成堆溢出。
  3. free：释放指定的索引对于的空间。
  4. Useless：没有任何用处，也没有输出任何东西。

利用过程

利用思路

首先分配三个chunk，溢出chunk2，然后释放chunk3，使得chunk2和chunk3合并，触发unlink，然后将free@got的内容改成put@plt，泄漏libc地址，最后将atoi@got的内容改写成system的地址，输出”/bin/sh”即可getshell。

1. 分配3个chunk,溢出chunk2。

可以看到chunk中已经有一个fake chunk，chunk3的prev_size域被覆盖为0x30，size域被覆盖为0x90，P标识位为0，表示上一个chunk处于free状态。

2. Free (3)，触发unlink，使得chunk2和chunk3合并

Free() chunk3之后，由于chunk3的prev_size域为0x30，并且P标志位为0，所以会触发unlink合并chunk3和fake chunk 合并。使得0x602150 = 0x602138。此时chunk2的地址为0x602138。

计算过程：

此时global[2]即chunk2的地址已经被改成0x602138。

3. 编辑global[2]，global[0] = free@got ; global[1] = puts@got ; global[2] = atoi@got

编辑chunk2，此时chunk2的地址已经指向0x602138的位置，即可以编辑chunk2，将global[0] = free@got 、 global[1] = puts@got 、global[2] = atoi@got。

4. 将free@got指针的内容替换成puts@plt

这样下次调用free的时候就调用的是puts。

5. Free(1) 泄漏libc地址

这是调用free(1)实际调用的是puts(global[1])，此时global[1]已经被覆盖为put@got，所以会输出puts函数的真实地址。

6. 覆盖atoi@got = system_addr

下一次调用atoi函数的时候就会调用system函数。
