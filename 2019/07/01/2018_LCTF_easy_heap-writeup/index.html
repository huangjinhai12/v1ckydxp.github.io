2018_LCTF_easy_heap writeup

Posted on 2019-07-01 | Post modified: 2019-07-11 | In 二进制 |
Words count in article: 2.7k | Reading time ≈ 12

这两天开始看tcache的一些利用方式，边做题边理解。当malloc()时，申请的大小正好与unsorted bin中的chunk大小一致时，会被放入tcache。由于tcache只利用前8字节保存链表信息，因此从unsorted bin中被放入tcache的chunk的bk的值仍然被保留。利用堆块中残留的bk指针的信息，进行unlink攻击。

这里有两点需要注意放在前面：

1、off by one可以覆盖inuse位：必须在进行了重用的情况下才能实现！

2、泄露堆地址时，加减的偏移量应取多少，需要考虑是否有重用！

chunk的fd指针和bk指针一直分不清，fd指针应该是指向该chunk的低地址的chunk，而bk指针是指向高地址的chunk。因为由当前chunk求下一个chunk的地址是当前chunk的地址加上当前chunk的size来得到的；而求前一个chunk的地址是由当前chunk的地址减去prev_size得到的。

程序基本信息

这道题的libc版本是2.27，所以在堆块申请释放的时候考虑Tcache，程序的所有的安全保护机制默认开启。

程序逻辑

​ 程序逻辑功能很简单，malloc/free/puts/exit。

  1. malloc：根据输入的size，读取指定长度的内容，长度不能超过0xf8。
  2. Free：常规内存释放，free之后有将指针置为0.
  3. puts：输出知道索引的内容。
  4. exit：程序退出。

结构体

又一个结构体数组，每一个结构体包含一个指向content的指针和表示content长度的size变量。

getline()函数

程序中有一个自己实现的getline()函数，但这个存在一个null-bute-overflow的漏洞。

size - 1 < index这里条件不满足退出循环之后，ptr[size] = 0;这步操作会导致一个byte的null溢出。

利用思路

​ 由于上面getline()函数每次只读取一个字节，并且该字节不能为0，所以常规的null-byte-overflow的方法行不通。常规的null-byte-overflow的利用思路是通过溢出覆盖prev_inuse字节来使得堆块合并，然后用伪造的prev_size字节使得和fake chunk进行unlink。但是这里不能通过输入0x00的值，所以prev_size无法通过溢出修改。

​ 在无法使用手动修改prev_size的情况下，考虑使用系统写入prev_siz。因为unsorted bin在进合并的时候会写入prev_size，所以可以利用该prev_size利用。在libc 2.27中，加入了Tcache内存管理机制，这里固定分配堆块大小为0xF8，符合Tcache的大小限制，所以在释放该堆块时会首先放入Tcache结构体中，直到达到上限7。这里content_chunk数组的数量为10，我们可以先Tcache填满，之后释放的就会放入unsorted bin中。同样的，要拿出unsorted bin中的堆块，也要先malloc7次把Tcache清空才可以。

PS：根据堆块重用的规则，这里申请分配的堆块大小为0xf8，对齐之后实际大小为0x100，比原来多了8字节，由于在64位程序中，8字节大于等于16字节的一半，所以会采用堆块重用，0xf8的最后8字节会在下一个chunk的prev_size域，加起来刚好0x100。

总结一下堆块重用条件：申请的内存大小在按照上述规则进行向大取整后，得到的应有大小比原大小大出的值>=对齐字节量的一半（presize字段长度）. =>也即：申请的内存大小mod对齐字节量<=对齐字节量的一半（presize字段长度）.

Tcache转移机制：当用户申请一个堆块的时候，如果tcache已经为空，而fastbin/smallbin/unsorted bin中有size符合的chunk时，会先把fastbin/smallbin/unsorted bin中的chunk全部转移到tcache中直到填满，然后再从tcache中拿；

注意：unsorted bin合并时，合并后的堆块不会进tcache，在从一个大的unsorted bin chunk分割出chunk的情形下，分出来的和剩下的都不会进tcache！在进行unsorted bin合并时，会把合并后的堆块的大小写如下一个chunk的prev_size域。利用这个可以实现chunk overlapping。

####具体过程为：

  1. 将 A -> B -> C 三块 unsorted bin chunk 依次进行释放
  2. A 和 B 合并，此时 C 前的 prev_size 写入为 0x200。
  3. A 、 B 、 C 合并，步骤 2 中写入的 0x200 依然保持。
  4. 利用 unsorted bin 切分，分配出 A 。
  5. 利用 unsorted bin 切分，分配出 B(B被分配两次，就有两个指针都指向这个位置)，注意此时不要覆盖到之前的 0x200。
  6. 将 A 再次释放为 unsorted bin 的堆块，使得 fd 和 bk 为有效链表指针。
  7. 此时 C 前的 prev_size 依然为 0x200（未使用到的值），A B C 的情况： A (free) -> B (allocated) -> C (free)，如果使得 B 进行溢出，则可以将已分配的 B 块包含在合并后的释放状态 unsorted bin 块中。
